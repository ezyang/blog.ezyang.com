RES, written for Air Canada (but running into difficulties, including Air Canada trying to get rid of the project); branching out into different languages. Seeing things like components prototyped in Python and not performing adequately (blithe suggestion: rewrite in C++. Not an option!)

What's it like to be in a situation where you're learning so much domain specific knowledge about the airline industry, which you're pretty sure long-term you are not going to be interested in?

Scala and Clojure being used by some of the performance team: can afford to do this since we develop our tools in-house. Edward today would have no problem working on these; didn't manage to get to Scala and Clojure took some bit of ramp-up time; probably is kind of hard to find these types of engineers!
