[This Google tech talk about API design](http://www.youtube.com/watch?v=aAb7hSCtvGw) came to my attention, which got me thinking a little bit about the topic. The thesis of this essay is that making an interface statically safe forces a change in the API; the end result is generally perceived as harder to use.

------------------------------------------------------------------------

- Easy to learn
- Easy to use, even without documentation
- Hard to misuse
- Easy to read and maintain code that uses it

Tension between "hard to misuse" and "easy to learn"
