Inside the package, there are the build dependencies of the package. They expose a number of modules which are available for bytestring to use (but not available from the bytestring package itself). We've given these packages unspecified version numbers: when Cabal runs dependency resolution, it picks concrete instantiations of these packages (enforcing that for any given package name, the same instance is picked; this way, we can rely on the types being the same even if a package shows up multiple times).
